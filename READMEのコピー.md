# Studio et. / Maison nr. — Webサイト

## ファイル構成

```
studio-et/
├── index.html        ← メインサイト（公開用）
├── cms.html          ← コーディングレスCMSエディタ（非公開推奨）
├── site-data.js      ← コンテンツデータ（CMSで書き出し）
└── *.jpg             ← 写真ファイル（全て同じフォルダに置く）
```

## Google Cloud Storage でのホスティング手順

1. [Google Cloud Console](https://console.cloud.google.com/) にアクセス
2. Cloud Storage → バケットを作成
   - バケット名: 例 `studio-et-web`
   - リージョン: asia-northeast1 (東京)
3. バケット設定 → 「均一なバケットレベルのアクセス」を有効化
4. 「権限」→「プリンシパルを追加」→ `allUsers` に `Storage オブジェクト閲覧者` を付与
5. このフォルダ内の **全ファイル** をバケットにアップロード
6. `index.html` の「公開URL」をウェブサイトとして設定
   - バケット設定 → 「ウェブサイト設定」→ インデックスファイル: `index.html`
7. カスタムドメインを使う場合は Cloud Load Balancer + SSL を設定

## 写真の追加方法

1. 追加したい写真を `studio-et/` フォルダ（index.html と同じ場所）に配置
2. `cms.html` を開き、CMS 画面でギャラリーやヒーロー画像を差し替え
3. 「エクスポート」→「ダウンロード」で新しい `site-data.js` を取得
4. バケットに `site-data.js` と新しい写真ファイルをアップロード

## CMSの使い方

`cms.html` をブラウザで開くだけで使えます（ローカルでもOK）。

- **基本情報**: スタジオ名・タグライン・住所・電話番号などを編集
- **ヒーロー画像**: トップページのスライドショー画像を3枚選択
- **ギャラリー**: 写真の追加・削除・カテゴリ変更
- **プラン・料金**: 撮影プランの追加・編集・削除
- **エクスポート**: 編集内容を `site-data.js` として書き出してダウンロード

CMS はコーディング不要で、`site-data.js` を差し替えるだけでサイトに反映されます。

## 注意

- `cms.html` は非公開のままにすることを推奨します（パスワード保護などを検討）
- 写真はすべて `index.html` と同じフォルダに置いてください
